# AI, ML & Data Portfolio Project Checklist

A good portfolio project should prove you can work with messy data, explain your thinking, and connect results to a real problem.

## Good project types

- Churn prediction
- Fraud detection
- Resume/job matching
- Salary analysis
- Customer segmentation
- Product analytics dashboard
- Stock or market analysis
- NLP text classifier
- Image classifier
- Recommendation system
- Forecasting model
- A/B test analysis
- RAG application (chat over a doc set)
- Fine-tuned LLM for a narrow task
- Vector search demo

## Project README checklist

Each project README should include:

- Problem statement
- Dataset source (and a license note)
- Tools used
- Steps taken
- Key charts
- Model or analysis result
- What you learned
- What you would improve
- How to run the project

## Strong project structure

```text
project-name/
  README.md
  data/          (or a script to download data)
  notebooks/
  src/
  charts/
  requirements.txt
```

## What recruiters like to see

- Clear goal
- Clean charts
- Simple explanation
- Real tradeoffs
- Business impact
- Code that actually runs
- Honest limits

## What to avoid

- No explanation
- Broken notebooks
- Fake metrics
- Copy-pasted Kaggle projects with no changes
- Huge files pushed directly to GitHub (use Git LFS or external storage)
- No link to the final result

## Portfolio site tips

- 3 to 5 strong projects beat 15 weak ones.
- Add a one-line problem statement above each project.
- Show one chart or screenshot per project.
- Link to the GitHub repo and (if applicable) a live demo.
- Keep your resume link and contact email easy to find.
